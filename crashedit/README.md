# CrashEdit - Complete Usage Guide

*(For Spanish version, see [README_ESP.md](README_ESP.md))*

## Introduction

CrashEdit is an FTN (FidoNet) message reader and editor that allows you to read message areas (echomail), private mail (netmail), write new messages, and reply to existing ones. It works internally in UTF-8.

---

## Starting the Program

### 1. Launch

When starting CrashEdit, the program looks for a configuration file:

- If you run `./crashedit` without arguments, it looks for `crashedit.conf` in the current directory
- If you specify a file: `./crashedit /path/to/myconf.conf`

**First run:** If no configuration exists, the setup screen opens automatically where you must define:
- Your name (SYSOP)
- At least one AKA address (your FidoNet address) (those you have in the golded area, which you can use in netmail)
- The area file (AREAFILE) generated by crashexport

### Initial Screen: Area List

Once configured, CrashEdit displays the **Area List** - all available message areas organized in a table.
If you cannot access them, verify that the tosser has created the files.

**What you see:**
- Area number, short name (tag), description
- Total messages and unread new messages

**What you can do:**
- Navigate with up/down arrows
- Search area by name with `/`
- Change order with `s`
- Enter an area with `Enter`

---

## Writing a New Message

The **Editor** opens with the cursor in the "To" field:

### Filling in the Header Fields

**Navigation between fields:**
- `Tab` / `↓` / `Ctrl+N`: Next field
- `↑` / `Ctrl+P`: Previous field
- `Enter` in any header field: Go directly to body
- `Ctrl+N` or `Ctrl+P` when in body: Return to header

1. **To:** Type the recipient's name or `All` for everyone
2. **Subj (Subject):** Type the message subject
3. Press `Enter` or `Tab` to go to the body

**Note for Netmail:**
- `Ctrl+A` or `F4`: Change AKA (sending address). Displays list of AKAs configured in areas.golded

### Writing the Body and Saving

Use the **Editor** keys (see "Editor Usage" section below) to write your message.

When finished, press **`F2`** or **`Ctrl+S`** to save.

**Note:** CrashEdit saves the message but does not send it directly. To prepare for sending, use the tosser and then the mailer will do its part.

## Netmail Special Functions

### CC (Carbon Copy) - Copies to Multiple Recipients

To send copies of a message to multiple recipients, write `CC:` lines in the message body:

```
CC: 2:341/207@fidonet
CC: 39:190/101@amiganet
CC: Blabla 2:341/201@fidonet 
```

**Format:**
- **Address only:** `CC: 2:341/207@fidonet` → Copy goes to the same name as the **To** field but at that FTN address
- **With name:** `CC: Blabla 2:341/201@fidonet` → Copy goes to **Blabla** at that FTN address

CrashEdit will convert these lines into `^ACC:` kludges and the message will be sent to all specified recipients. This is useful for sending the same message to several nodes without writing separate messages.

### File Request (FREQ) - Requesting Files via FidoNet

From the area or message list, press **`Ctrl+F`** to open the File Request screen.

**Steps:**
1. Enter the FTN address of the node that has the files
2. Select the outbound mode:
   - **ASO** (flat): All `.req` files in a single directory
   - **BSO** (BinkleyStyle): Organized by zones
   - **BSO+ext**: BSO with hexadecimal zone extension
3. Add files to request with **`A`**
4. (Optional) Set password with **`P`**
5. Generate the request with **`W`** (write)

**Requirement:** You must have `OUTBOUND` configured in your `crashedit.conf` pointing to your mailer's outbound directory.

The generated `.req` file will be processed by your mailer and sent to the destination node, which will respond by sending the requested files.

### File Attachments (File Attach)

In **netmail** messages, you can attach files to send them to other nodes:

**In the editor:**
- **`Ctrl+F`**: Add file attachment (select file from disk)
- **`Ctrl+Q`**: Remove selected attachment
- **`Ctrl+L`**: List all attachments
- **`Alt+L`**: Clear all attachments

**Important notes:**
- Maximum 32 file attachments per message
- File names appear in the subject line (limited to 71 characters by FTS-1)
- If there are many files, the message is automatically split into several
- Attachments are sent as messages with the `FileAttach` attribute

**To receive file attachments:**
The mailer will receive the files and place them in your inbound directory. The message in CrashEdit will display the file names in the subject.

## Message Reader

When entering an area and selecting a message, the **Reader** opens where you can see the full content.

### Navigation in the Reader

- **`↑` / `↓`** or **`PgUp`** / **`PgDn`**: Move through the text
- **`Space`**: Next page
- **`Home`**: Go to start of message
- **`End`**: Go to end
- **`→`** / **`←`**: Change message (previous/next)
- **`Enter`**: Return to message list

### ANSI Mode (Colors)

Messages may contain ANSI color codes (escape codes for colors and effects).

- **`a`**: Toggle ANSI mode (enable/disable colors)
- When ANSI is **enabled**: Color codes are interpreted and text is displayed with colors
- When ANSI is **disabled**: Codes are displayed as plain text (e.g.: `[0;32m`)

This is useful if a message has ANSI codes that make reading difficult or if you want to see the text without formatting.

### Show/Hide Elements

- **`k`**: Show/hide kludges (control lines like ^A MSGID, SEEN-BY, Via)
- **`v`**: Toggle view of kludges and hidden lines

### Change Encoding (Charset)

- **`c`**: Change display charset (LATIN-1, CP437, UTF-8, etc.)
- Allows viewing messages written in different encodings correctly

### Encoding Modes (Charsets)

CrashEdit supports multiple character encodings for FTN messages:

**Available Charsets:**
- **CP437** (default for reading): Codepage 437, traditionally used in DOS/FTN
- **CP850**: Codepage 850 (multilingual DOS)
- **CP865**: Codepage 865 (Nordic)
- **CP866**: Codepage 866 (Cyrillic)
- **LATIN-1** (ISO-8859-1): Western European
- **LATIN-2** (ISO-8859-2): Central European
- **CP1252**: Windows-1252
- **UTF-8** (default for writing): Unicode, full character support

**AUTO Mode (Display):**
- When the display charset is set to **Auto**, CrashEdit automatically detects the message encoding
- It looks for the `^ACHRS:` or `^ACHARSET:` kludge in the message
- If no kludge is found, it defaults to CP437
- This allows reading old messages automatically without manual configuration

**Two Independent Charsets:**

1. **View Charset (Display)**: How the message is shown on screen
   - Auto = detect from message
   - Manual = force a specific encoding

2. **Save/Output Charset (Saving)**: How messages you write are saved
   - Configurable in Setup
   - UTF-8 by default for maximum modern compatibility
   - You can change it to CP437 for compatibility with old systems

**When to use Manual mode:**
- If a message looks wrong (strange characters), try manually changing to LATIN-1 or CP850
- For messages in Cyrillic, use CP866
- If Auto mode fails to detect correctly

## Replying to a Message

In the reader, press **`r`** (reply).

The **Editor** opens with pre-filled data:
- The subject is copied preceded by "Re:"
- The recipient is copied from the original sender
- The original text is quoted with `>` at the beginning of each line

Write your reply below the quoted text and press **`F2`** or **`Ctrl+S`** to save.

---

## Editing Your Own Message

Navigate to a message you wrote (your name appears in "From") and press **`e`** (edit).

The **Editor** opens with the message loaded. Make the necessary changes and save with **`F2`** or **`Ctrl+S`**.

**Note:** You can only edit your own messages. The program verifies that the sender matches your configured AKA.

---

## Deleting Messages

### Option A: From the Message List
1. Select the message with the arrows
2. Press **`d`** (delete)
3. Confirm with `Y` when prompted

### Option B: From the Reader
1. Read the message
2. Press **`d`** or **`Del`**
3. Confirm with `Y`

**The message is not physically deleted** - it is marked for deletion and will be removed during message database maintenance (purge).

---

## Catch-up (Mark as Read)

When you have read what interests you from an area and want to mark the rest as read:

### From the Area List
1. Select the area
2. Press **`C`** or **`Alt+C`**
3. Select `Y` for this area or `A` for all

### From the Message List
Press **`C`** - it will mark all messages in the current area as read.

---

## Message Search

### Quick Search (From/To/Subj)
From the message list:
1. Press **`/`**
2. Type the text to search
3. Use `n` for next match

### Complete Search (Headers + Body)
From the message list or areas:
1. Press **`P`**
2. The search popup appears:
3. Configure options:
   - `Space` on a checkbox to enable/disable
   - Search pattern required
4. `Enter` to execute

**Results:** A list of areas with matches is displayed. Select an area to view the found messages.

## Editor Usage

The editor is used to write and modify messages. Access it from:
- **Write new message**: From the area list with `N` or `Ins`
- **Reply**: From the reader with `r`
- **Edit own message**: From the message list or reader with `e`

### Basic Editing

- **`Enter`**: New line
- **`Backspace`** / **`Delete`**: Delete characters
- **`Ctrl+K`** / **`Ctrl+Y`**: Delete entire line
- **`Ctrl+Z`**: Undo last change
- **`Alt+Z`**: Redo
- **`F2`** / **`Ctrl+S`**: Save message
- **`F10`** / **`Esc`** / **`Ctrl+Q`**: Cancel editing

### Navigation Between Fields (Header)

- **`Tab`** / **`↓`** / **`Ctrl+N`**: Next field (To → Subj → Body)
- **`↑`** / **`Ctrl+P`**: Previous field
- **`Enter`** in header: Go to body directly
- **`Ctrl+N`** or **`Ctrl+P`** when in body: Return to header

### Selection and Blocks (F6)

**Mark a block:**
1. Place the cursor where the selection starts
2. Press **`F6`** - active block indicator appears
3. Move the cursor to the end of the selection
4. The text between both positions is highlighted

**Block operations:**
- **`Ctrl+C`**: Copy block (also to system clipboard)
- **`Ctrl+X`**: Cut block (also to clipboard)
- **`Ctrl+V`**: Paste block (or from clipboard if no internal block)
- **`Delete`** or **`Backspace`**: Delete selected block
- **`Ctrl+O`**: Export block to file

### System Clipboard

CrashEdit integrates with the operating system clipboard:

- **Copy (`Ctrl+C`)**: The selected text is copied to both the internal clipboard and the system clipboard. You can paste it in other applications.
- **Cut (`Ctrl+X`)**: Same as copy, but deletes the text from the editor.
- **Paste (`Ctrl+V`)**: First tries to paste from the internal clipboard (block marked with F6). If there is no internal block, it pastes from the system clipboard.

This allows exchanging text between CrashEdit and other applications (browser, text editor, etc.).

### Search and Replace

- **`F5`**: Search text forward
- **`Ctrl+R`**: Interactive search and replace
- **`Ctrl+G`**: Go to specific line number

### Undo/Redo

- **`Ctrl+Z`**: Undo last change
- **`Alt+Z`**: Redo

### Adjust Paragraph

- **`Ctrl+W`**: Reformat paragraph to 75 columns (useful for quotes)

### Insert File

- **`F7`**: Insert content from a file on disk

### Kludges (FTN Control Lines)

- **`F8`**: View and edit control lines (^A MSGID, REPLY, etc.)

### Attachments (Netmail Only)

- **`Ctrl+F`**: Add file attachment
- **`Ctrl+Q`**: Remove attachment
- **`Ctrl+L`**: List attachments
- **`Alt+L`**: Clear all attachments

## Configuration (Setup)

Access configuration from any main screen with **`S`**.

### Configurable Options

**Personal Data:**
- SysOp name
- AKA addresses (multiple)

**Files:**
- AREAFILE (area list)
- Default charset

**Preferences:**
- Character encoding (LATIN-1, CP437, UTF-8)
- Origin for messages
- Max hits in searches

**Display Options:**
- Colors and theme
- Display margin

---

## Closing the Program

### From the Area List
Press **`q`**. If there are no pending operations, the program closes immediately.

### From Any Other Screen
Press **`q`** or **`Esc`** repeatedly until you return to the area list, then **`q`** to exit.

### From the Editor
- **`F10`**, **`Esc`**, or **`Ctrl+Q`**: Cancel editing
- If there are unsaved changes, confirmation will be requested:
  - `Y` = Discard changes and exit
  - `N` = Return to editor

---

## Useful Tips

1. **Always use `?` or `F1`** on any screen to see available keys
2. **Frequent catch-up**: Use `C` regularly to keep the new message counter under control
3. **Charset**: If you see strange characters, use `Alt+H` in the editor to change encoding (F3 also works but has contextual behavior in search mode)
4. **Search**: Use `P` to search in message bodies (slower but complete)
