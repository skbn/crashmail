# CrashEdit - Complete Usage Guide

## Introduction

CrashEdit is an FTN (FidoNet) message reader and editor that allows you to read message areas (echomail), private mail (netmail), write new messages, and reply to existing ones. It works internally in UTF-8.

---

## Program Startup

### 1. Launch

When starting CrashEdit, the program looks for a configuration file:

- If you run `./crashedit` without arguments, it looks for `crashedit.conf` in the current directory
- If you specify a file: `./crashedit /path/to/myconf.conf`

**First run:** If no configuration exists, the Setup screen opens automatically where you must define:
- Your name (SYSOP)
- At least one AKA address (your FidoNet address) (The ones you can use in netmail are those in the golded area)
- The area file (AREAFILE) generated by crashexport

### Initial Screen: Area List

Once configured, CrashEdit displays the **Area List** - all available message areas organized in a table.
If you cannot access them, verify that the tosser has created the files.

**What you see:**
- Area number, short name (tag), description
- Total messages and unread new messages

**What you can do:**
- Navigate with up/down arrows
- Search area by name with `/`
- Change sort order with `s`
- Enter an area with `Enter`

---

## Writing a New Message

The **Editor** opens with the cursor in the "To" field:

### Filling Header Fields

**Navigation between fields:**
- `Tab` / `↓` / `Ctrl+N`: Next field
- `↑` / `Ctrl+P`: Previous field
- `Enter` in any header field: Go directly to body
- `Ctrl+N` or `Ctrl+P` if in body: Return to header

1. **To (Para):** Enter recipient name or `All` for everyone
2. **Subj (Subject):** Enter message topic
3. Press `Enter` or `Tab` to go to body

**Note for Netmail:**
- **`Ctrl+A`** or **`F4`**: Change AKA (sending address). Shows list of AKAs configured in areas.golded
- **`F10`** / **`Alt+P`**: Nodelist picker - select recipient from nodelist for To field

### Nodelist Picker

The **Nodelist Picker** allows selecting recipients from loaded nodelists (configured with `INCLUDE` in crashedit.conf).

**Usage:**
- From the "To" field in the editor, press **`F10`** or **`Alt+P`**
- A list appears with all nodes from the nodelist
- You can type to filter by name or address (type-ahead)
- Press `Enter` to select the recipient
- The name and address are automatically copied to the "To" field

**Navigation in picker:**
- **`↑`** / **`↓`**: Navigate list
- **`F10`**: Select recipient
- **`Ctrl+F10`**: Browse nodelist without selecting (browse mode)
- **`Esc`**: Cancel

### Writing Body and Saving

Use the **Editor** keys (see "Editor Usage" section below) to write your message.

When finished, press **`F2`** or **`Ctrl+S`** to save.

**Note:** CrashEdit saves the message but does not send it directly. Use the tosser to prepare sending, then the mailer will handle it.

## Netmail Special Functions

### CC (Carbon Copy) - Multiple Recipients

To send copies of a message to multiple recipients, write `CC:` lines in the message body:

```
CC: 2:341/207@fidonet
CC: 39:190/101@amiganet
CC: Blabla 2:341/201@fidonet 
```

**Format:**
- **Address only:** `CC: 2:341/207@fidonet` → Copy goes to same name in **To** field but at that FTN address
- **With name:** `CC: Blabla 2:341/201@fidonet` → Copy goes to **Blabla** at that FTN address

CrashEdit will convert these lines to `^ACC:` kludges and the message will be sent to all specified recipients. This is useful for sending the same message to multiple nodes without writing separate messages.

### File Request (FREQ) - Request Files via FidoNet

From the area or message list, press **`Ctrl+F`** to open the File Request screen.

**Steps:**
1. Enter the FTN address of the node that has the files
2. Select outbound mode:
   - **ASO** (flat): All `.req` files in a single directory
   - **BSO** (BinkleyStyle): Organized by zones
   - **BSO+ext**: BSO with hexadecimal zone extension
3. Add files to request with **`A`**
4. (Optional) Set password with **`P`**
5. Generate request with **`W`** (write)

**Requirement:** You must have `OUTBOUND` configured in your `crashedit.conf` pointing to your mailer's outbound directory.

The generated `.req` file will be processed by your mailer and sent to the destination node, which will respond by sending the requested files.

### File Attachments

In **netmail** messages, you can attach files to send to other nodes:

**In the editor:**
- **`Ctrl+F`**: Add file attachment (select file from disk)
- **`Ctrl+Q`**: Remove selected attachment
- **`Ctrl+L`**: List all attachments
- **`Alt+L`**: Clear all attachments

**Important notes:**
- Maximum 32 file attachments per message
- File names appear in the subject line (limited to 71 characters by FTS-1)
- If there are many files, the message is automatically split into multiple messages
- Attachments are sent as messages with the `FileAttach` attribute

**To receive file attachments:**
The mailer will receive the files and place them in your inbound directory. The message in CrashEdit will show the file names in the subject.

## Message List

When you enter an area (by pressing `Enter` in the area list), the **Message List** for that area appears.

**What you see:**
- Message number
- Date
- Sender (From)
- Recipient (To)
- Subject
- Indicator if marked for deletion

**What you can do:**
- **`↑`** / **`↓`**: Navigate between messages
- **`Enter`**: Read selected message
- **`N`** / **`Ins`**: Write new message
- **`r`**: Reply to selected message
- **`e`**: Edit own message
- **`d`**: Delete message
- **`C`**: Mark all as read (catch-up)
- **`/`**: Quick search (From/To/Subj)
- **`P`**: Full search (headers + body)
- **`Ctrl+F`**: File Request
- **`q`**: Return to area list

## Message Reader

When entering an area and selecting a message, the **Reader** opens where you can see the complete content.

### Reader Navigation

- **`↑`** / **`↓`** or **`PgUp`** / **`PgDn`**: Move through text
- **`Space`**: Next page
- **`Home`**: Go to message start
- **`End`**: Go to message end
- **`→`** / **`←`**: Change message (previous/next)
- **`Enter`**: Return to message list

### ANSI Mode (Colors)

Messages may contain ANSI color codes (escape codes for colors and effects).

- **`a`**: Toggle ANSI mode (enable/disable colors)
- When ANSI is **active**: Color codes are interpreted and text is shown with colors
- When ANSI is **inactive**: Color codes are shown as plain text (e.g: `[0;32m`)

This is useful if a message has ANSI codes that make reading difficult or if you want to see the text without formatting.

### Show/Hide Elements

- **`k`**: Show/hide kludges (control lines like ^A MSGID, SEEN-BY, Via)
- **`v`**: Toggle kludge and hidden line view

### Change Charset

- **`c`**: Change display charset (LATIN-1, CP437, UTF-8, etc.)
- Allows viewing messages written in different encodings correctly

### Charset Modes

CrashEdit supports multiple character encodings for FTN messages:

**Available charsets:**
- **CP437** (default for reading): Codepage 437, traditionally used in DOS/FTN
- **CP850**: Codepage 850 (multilingual DOS)
- **CP865**: Codepage 865 (Nordic)
- **CP866**: Codepage 866 (Cyrillic)
- **LATIN-1** (ISO-8859-1): Western European
- **LATIN-2** (ISO-8859-2): Central European
- **CP1252**: Windows-1252
- **UTF-8** (default for writing): Unicode, full character support

**AUTO Mode (Display):**
- When display charset is in **Auto**, CrashEdit automatically detects message encoding
- Looks for `^ACHRS:` or `^ACHARSET:` kludge in the message
- If no kludge found, uses CP437 by default
- This allows reading old messages automatically without manual configuration

**Two Independent Charsets:**

1. **View Charset (Display)**: How the message is shown on screen
   - Auto = detect from message
   - Manual = force specific encoding

2. **Save/Output Charset (Saving)**: How messages you write are saved
   - Configurable in Setup
   - UTF-8 by default for maximum modern compatibility
   - Can change to CP437 for compatibility with old systems

**When to use Manual mode:**
- If a message looks wrong (strange characters), try changing manually to LATIN-1 or CP850
- For Cyrillic messages, use CP866
- If Auto mode fails to detect correctly

## Replying to a Message

In the reader, press **`r`** (reply).

The **Editor** opens with pre-filled data:
- Subject is copied prefixed with "Re:"
- Recipient is copied from original sender
- Original text is quoted with `>` at the start of each line

Write your response below the quoted text and press **`F2`** or **`Ctrl+S`** to save.

---

## Editing Your Own Message

Navigate to a message you wrote (your name appears in "From") and press **`e`** (edit).

The **Editor** opens with the message loaded. Make necessary changes and save with **`F2`** or **`Ctrl+S`**.

**Note:** You can only edit your own messages. The program verifies that the sender matches your configured AKA.

---

## Deleting Messages

### Option A: From Message List
1. Select the message with arrows
2. Press **`d`** (delete)
3. Confirm with `Y` when prompted

### Option B: From Reader
1. Read the message
2. Press **`d`** or **`Del`**
3. Confirm with `Y`

**The message is not physically deleted** - it is marked for deletion and will be removed during message base maintenance (purge).

---

## Catch-up (Mark as Read)

When you have read what interests you from an area and want to mark the rest as read:

### From Area List
1. Select the area
2. Press **`C`** or **`Alt+C`**
3. Select `Y` for this area or `A` for all

### From Message List
Press **`C`** - will mark all messages in current area as read.

---

## Message Search

### Quick Search (From/To/Subj)
From the message list:
1. Press **`/`**
2. Enter text to search
3. Use `n` for next match

### Full Search (Headers + Body)
From the message list or areas:
1. Press **`P`**
2. Search popup appears:
3. Configure options:
   - `Space` on a checkbox to enable/disable
   - Search pattern is required
4. `Enter` to execute

**Results:** Shows area list with matches. Select an area to see found messages.

## Editor Usage

The editor is used to write and modify messages. Accessed from:
- **Write new message**: From area list with `N` or `Ins`
- **Reply**: From reader with `r`
- **Edit own message**: From message list or reader with `e`

### Basic Editing

- **`Enter`**: New line
- **`Backspace`** / **`Delete`**: Delete characters
- **`Ctrl+K`** / **`Ctrl+Y`**: Delete entire line
- **`Ctrl+Z`**: Undo last change
- **`Alt+Z`**: Redo
- **`F2`** / **`Ctrl+S`**: Save message
- **`F10`** / **`Esc`** / **`Ctrl+Q`**: Cancel editing

### Header Field Navigation

- **`Tab`** / **`↓`** / **`Ctrl+N`**: Next field (To → Subj → Body)
- **`↑`** / **`Ctrl+P`**: Previous field
- **`Enter`** in header: Go directly to body
- **`Ctrl+N`** or **`Ctrl+P`** while in body: Return to header

### Selection and Blocks

**Word-by-word selection:**
- **`Ctrl+Shift+Left/Right`**: Select word by word horizontally
- **`Ctrl+Shift+Up/Down`**: Select line by line vertically

**Selection operations:**
- **`Ctrl+C`**: Copy selection (also to system clipboard)
- **`Ctrl+X`**: Cut selection (also to system clipboard)
- **`Ctrl+V`**: Paste (or from system clipboard if no internal selection)
- **`Delete` or `Backspace`**: Delete selection
- **`Ctrl+O`**: Export selection to file

### System Clipboard

CrashEdit integrates with the system clipboard:

- **Copy (`Ctrl+C`)**: Selected text is copied to both internal and system clipboard. You can paste it in other applications.
- **Cut (`Ctrl+X`)**: Same as copy, but deletes text from editor.
- **Paste (`Ctrl+V`)**: First tries to paste from internal clipboard. If no internal selection, pastes from system clipboard.

This allows exchanging text between CrashEdit and other applications (browser, text editor, etc.).

### Search and Replace

- **`F5`**: Search text forward
- **`Ctrl+R`**: Interactive search and replace
- **`F6`** / **`Alt+B`**: Replace all occurrences (search mode)
- **`Ctrl+G`**: Go to specific line number

### Undo/Redo

- **`Ctrl+Z`**: Undo last change
- **`Alt+Z`**: Redo

### Reformat Paragraph

- **`Ctrl+W`**: Reformat paragraph to 75 columns (useful for quotes)

### Insert File

- **`F7`**: Insert file content from disk

### Kludges (FTN Control Lines)

- **`F8`**: View and edit control lines (^A MSGID, REPLY, etc.)

### Attachments (Netmail Only)

- **`Ctrl+F`**: Add file attachment
- **`Ctrl+Q`**: Remove attachment
- **`Ctrl+L`**: List attachments
- **`Alt+L`**: Clear all attachments

### Spell Checker

- **`Alt+E`**: Show/hide spell panel
- **`Alt+T`**: Enable/disable spell checking
- **`Alt+W`**: Check word under cursor

**Configuration:**
These options are configured from **Setup** (key `S`):
- Spell Checker Enabled: Enable/disable spell checker
- Spell Dictionary Path: Directory with Hunspell dictionaries (.aff/.dic)
- Spell Dictionary Name: Dictionary name (e.g: en_US, es_ES)
- Spell Custom Dictionary: Custom dictionary file

### Hyphenation

- **`Alt+L`**: Enable/disable hyphenation in hard-wrap (hard-wrap mode only)

**Configuration:**
These options are configured from **Setup** (key `S`):
- Hyphenation Enabled: Enable/disable hyphenation
- Hyphenation Dictionary Path: Directory with hyphenation dictionaries (hyph_*.dic)
- Hyphenation Dictionary Name: Dictionary name (e.g: en_US, es_ES)
- Hyphenation Wrap Enabled: Enable hyphenation at wrap

### Thesaurus (Synonyms)

- **`Alt+J`**: Look up synonyms for word under cursor

**Configuration:**
These options are configured from **Setup** (key `S`):
- Thesaurus Enabled: Enable/disable thesaurus
- Thesaurus Path: Directory with thesaurus dictionaries (th_*.idx/.dat)
- Thesaurus Dictionary: Dictionary name (e.g: en_US_v2, es_v2)

### Glyph Picker (Unicode Character Selector)

- **`Alt+U`**: Open Unicode character selector to insert special symbols

## Configuration (Setup)

Access configuration from any main screen with **`S`**.

### Configurable Options

**Personal Data:**
- SysOp name
- AKA addresses (multiple)

**Files:**
- AREAFILE (area list)
- Default charset

**Preferences:**
- Character encoding (LATIN-1, CP437, UTF-8)
- Origin (for messages)
- Max hits in searches

**Display Options:**
- Colors and theme
- Display margin

## Program Exit

### From Area List
Press **`q`**. If no pending operations, the program closes immediately.

### From Any Other Screen
Press **`q`** or **`Esc`** repeatedly until returning to area list, then **`q`** to exit.

### From Editor
- **`F10`**, **`Esc`**, or **`Ctrl+Q`**: Cancel editing
- If there are unsaved changes, confirmation is requested:
  - `Y` = Discard changes and exit
  - `N` = Return to editor

---

## Useful Tips

1. **Always use `?` or `F1`** in any screen to see available keys
2. **Frequent catch-up**: Use `C` regularly to keep new message count under control
3. **Charset**: If you see strange characters, use `Alt+H` in editor to change encoding (F3 also works but has contextual behavior in search mode)
4. **Search**: Use `P` to search in message body (slower but complete)
